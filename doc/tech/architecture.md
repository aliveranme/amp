## 组件说明

### 1. Agent 引擎

四种模式对应不同的模型和推理预算：

| 模式 | 主模型 | 用途 |
|------|--------|------|
| `low` | 快速模型 | 小型确定任务，低消耗 |
| `medium` | 平衡模型 | 大部分任务 |
| `high` | GPT-5.6 Sol (deep reasoning) | 困难任务 |
| `ultra` | 最强组合 | 开放式困难任务 |

子 Agent 管理：自动为复杂任务生成独立 Agent 实例，主要工作在 `medium` 模式。

### 2. 内置工具

| 工具 | 功能 | 底层模型 |
|------|------|----------|
| **Oracle** | 第二意见/深度分析 | GPT-5.6 Sol 或 Claude Fable 5 |
| **Librarian** | 远程代码搜索 | GitHub API 集成 |
| **Painter** | 图片生成/编辑 | GPT Image 2 |
| **Subagents** | 并行任务执行 | 继承当前模式 |

### 3. Plugin 系统

TypeScript 插件框架，基于 `@ampcode/plugin` 类型包：

```
session.start → agent.start → [tool.call → tool.result] → agent.end
```

关键能力：
- `registerTool()` — 注册自定义工具
- `registerCommand()` — 注册命令面板命令
- `amp.on()` — 事件监听
- `ctx.ui.*` — 用户界面交互
- `amp.ai.ask()` — AI 分类决策

### 4. MCP 框架

支持本地和远程 MCP 服务器：

- **本地**: 通过 `command + args` 启动子进程
- **远程**: 通过 SSE 协议连接
- **OAuth**: 对远程服务器支持自动 OAuth 2.0 流程
- **权限**: `amp.mcpPermissions` 允许/拒绝规则
- **Skill 内嵌**: mcp.json 放置于 skill 目录中

### 5. 远程执行 (Orbs / Runners)

- **Orbs**: 云端持续运行的 amp 实例，关闭终端后继续工作
- **Runners**: 在本地机器上以 `--no-tui` 模式运行，接受远程线程创建

## 数据流

```
用户提示词
    │
    ▼
Agent 引擎解析
    │
    ├─→ 调用工具（本地 Shell / MCP / Plugin）
    ├─→ 生成子 Agent（并行执行）
    ├─→ 调用 Oracle（第二意见）
    └─→ 调用 Librarian（代码搜索）
    │
    ▼
结果汇总 → 输出响应 → 继续/结束线程
```

## 关键协议

| 协议 | 端点 | 用途 |
|------|------|------|
| HTTPS | `ampcode.com/api/*` | 核心 API |
| OAuth 2.0 | `auth.openai.com/oauth/*` | BYOK 认证 |
| REST | `api.github.com` | 代码搜索 |
| MCP (SSE) | 用户配置 | 外部工具集成 |
| SSH | 用户配置 | Orb 远程执行 |
